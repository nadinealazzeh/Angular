الفكرة الأساسية

كل Component تحتاج ثلاثة أجزاء رئيسية: Class مكتوبة بـ TypeScript، Template تتحكم بما يظهر في الصفحة، وSelector يحدد كيف نستدعيها داخل HTML.

المثال الأساسي

profile-photo.ts
import { Component } from '@angular/core';

@Component({
  selector: 'profile-photo',
  template: `<img src="profile-photo.jpg"
                  alt="Your profile photo" />`,
})
export class ProfilePhoto {}
اقرئيه كجملة: يا Angular، هذا Class اسمه ProfilePhoto هو Component، ويتم استخدامه من خلال العنصر <profile-photo>.
ما هي Metadata؟
الـ Object الموجود داخل @Component({...}) يسمى Metadata، لأنه يصف للـ Angular كيف تعمل الـ Component: اسمها في HTML، قالبها، Styles الخاصة بها، وما الذي تستورده.
Inline أم ملفات منفصلة؟
ملفات منفصلة
@Component({
  selector: 'profile-photo',
  templateUrl: './profile-photo.html',
  styleUrl: './profile-photo.css',
})
export class ProfilePhoto {}

للمكونات الصغيرة يمكن استخدام template وstyles مباشرة. للمكونات الأكبر تكون الملفات المنفصلة أوضح وأسهل للصيانة.

Accepting Data with Inputs

ما المشكلة التي تحلها Inputs؟

بدون Input، ستضطرين لكتابة البيانات ثابتة داخل Child Component. هذا يجعلها غير قابلة لإعادة الاستخدام.

مثال: نفس UserCard قد تعرض Nadine مرة وSara مرة أخرى. لذلك الاسم يجب أن يأتي من الأب بدل أن يكون مكتوبًا داخل البطاقة.
الابن يعلن ما الذي يستطيع استقباله باستخدام input().
الأب يرسل القيمة عبر Property Binding.
Angular تحدث InputSignal عند تغير قيمة الأب.
الابن يقرأ القيمة باستخدام الأقواس ().

الفرق بين النص والمتغير

name="Nadine" يرسل نصًا ثابتًا. أما [name]="userName" فيرسل قيمة متغير موجود في Class الأب.

تطبيق

أنشئي Input اسمها title بقيمة افتراضية Untitled، ثم اعرضيها داخل Template.

المشكلة التي يحلها Style Encapsulation

في CSS العادي، قاعدة مثل button { color:red } قد تؤثر على كل أزرار الموقع. Angular تحاول إبقاء CSS كل Component قريبًا منها حتى لا يفسد بقية الواجهة.

تخيلي: عندك زر أحمر داخل Login وزر أخضر داخل Dashboard. بدون عزل جيد قد تتداخل القواعد.
تكتبين CSS خاصًا بالـ Component.
Angular تربطه بعناصر Template نفسها في وضع Emulated.
Global styles ما زالت قادرة على التأثير من الخارج.
Specificity وقواعد CSS العادية ما زالت مهمة.

جرّبي

اكتبي قاعدة بـ :host تجعل الـ Component نفسها display:block، ثم قاعدة ثانية لصورة داخلها.

لماذا نحتاج Selector؟

Angular تحتاج طريقة تربط بين Class في TypeScript وبين مكان استخدامها داخل HTML. الـ selector هو هذا الرابط.

اقرئيها هكذا:

selector: 'app-user-card' تعني: عندما ترى Angular العنصر <app-user-card>، أنشئي نسخة من Component المرتبطة به.

Angular تقرأ Template وقت الـ compile.
تبحث عن عناصر تطابق selectors المعروفة.
تنشئ Component instance لكل عنصر مطابق.
تعرض Template الخاصة بالـ Component داخله.

تطبيق ذهني

Component اسمها LessonCard. اكتبي selector مناسبًا لها ثم اكتبي طريقة استخدامها داخل HTML.

قبل الكود: ماذا تحاول Angular أن تحل؟

مثال واقعي: صفحة الملف الشخصي يمكن تقسيمها إلى Header، صورة شخصية، بيانات المستخدم، وقائمة النشاط. كل جزء يصبح Component مستقلة.

ما الفرق بين Input وOutput؟

Input تنقل البيانات من Parent إلى Child. Output تنقل حدثًا أو نتيجة من Child إلى Parent.

مثال: Parent يرسل اسم المنتج إلى ProductCard باستخدام Input، وعندما يضغط المستخدم زر الحذف ترسل ProductCard حدث deleted باستخدام Output.
الابن يعلن Output باستخدام output().
عند حدوث شيء، يستدعي emit().
الأب يستمع باستخدام الأقواس الدائرية.
أي بيانات مرسلة تصل داخل $event.

تطبيق

اعملي Output اسمها saved ترسل اسم المستخدم للأب عند الضغط على زر.

Child
emit() →
Parent Listener

الأساس

panel.ts
import {Component, output} from '@angular/core';

export class ExpandablePanel {
  panelClosed = output<void>();

  close() {
    this.panelClosed.emit();
  }
}
parent.html
<expandable-panel (panelClosed)="savePanelState()" />

بيانات

valueChanged.emit(7)

استقبال

(valueChanged)="logValue($event)"

Alias

output({alias: 'valueChanged'})

قديم

@Output() + EventEmitter
لا تعمل Bubble: Angular custom events لا تصعد عبر DOM تلقائيًا.

متى نحتاج Content Projection؟

عندما نريد Component تحدد الشكل الخارجي، لكن تسمح لمن يستخدمها أن يقرر المحتوى الداخلي.

الملكية المنطقية للمحتوى تبقى للأب.

لماذا نحتاج Query؟

أحيانًا لا نحتاج فقط عرض Child Component؛ نحتاج الوصول إلى instance نفسها أو DOM element أو TemplateRef.

مثال: لديك input داخل Template وتريدين عمل focus عليه، أو Child Component وتريدين استدعاء method منه.
حددي: هل العنصر داخل View أم داخل projected content؟
استخدمي viewChild أو contentChild.
النتيجة Signal وقد تكون undefined.
استخدمي required فقط عندما يكون وجود العنصر مضمونًا.

تطبيق

Content Queries

قاعدة Angular الأساسية

وصفي شكل DOM داخل Template، وغيّريه عبر Bindings. التعامل المباشر مع DOM استثناء وليس الأسلوب الأساسي.

لماذا؟ لأن Angular تحتاج أن تبقى هي المصدر الرئيسي لحالة الواجهة، ولأسباب الأداء والأمان وSSR.
استخدمي ElementRef للوصول إلى Host.
انتظري render callback قبل القياس أو focus.
استخدمي DOM API فقط للحالات التي لا توفرها Templates.

popup.ts
import {Component, computed, input, output} from '@angular/core';
import {animate, state, style, transition, trigger} from '@angular/animations';

@Component({
  selector: 'my-popup',
  template: `
    <span>Popup: {{ message() }}</span>
    <button type="button" (click)="closed.emit()">&#x2716;</button>
  `,
  animations: [
    trigger('state', [
      state('opened', style({transform: 'translateY(0%)'})),
      state('void, closed', style({transform: 'translateY(100%)', opacity: 0})),
      transition('* => *', animate('100ms ease-in')),
    ]),
  ],
  styles: [`:host { position:absolute; bottom:0; left:0; right:0; }`],
  host: {'[@state]': 'state()'},
})
export class Popup {
  readonly message = input('');
  readonly closed = output<void>();
  readonly state = computed(() => this.message() ? 'opened' : 'closed');
}
popup.service.ts
@Injectable()
export class PopupService {
  private injector = inject(EnvironmentInjector);
  private applicationRef = inject(ApplicationRef);

  showAsComponent(message: string) {
    const host = document.createElement('popup-component');
    const ref = createComponent(Popup, {
      environmentInjector: this.injector,
      hostElement: host,
    });
    this.applicationRef.attachView(ref.hostView);
    ref.instance.closed.subscribe(() => {
      document.body.removeChild(host);
      this.applicationRef.detachView(ref.hostView);
    });
    ref.setInput('message', message);
    document.body.appendChild(host);
  }

  showAsElement(message: string) {
    const popup = document.createElement('popup-element') as
      NgElement & WithProperties<Popup>;
    popup.addEventListener('closed', () =>
      document.body.removeChild(popup)
    );
    popup.setAttribute('message', message);
    document.body.appendChild(popup);
  }
}
app.ts
@Component({
  selector: 'app-root',
  template: `
    <input #input value="Message" />
    <button (click)="popup.showAsComponent(input.value)">
      Show as component
    </button>
    <button (click)="popup.showAsElement(input.value)">
      Show as element
    </button>
  `,
  providers: [PopupService],
  imports: [Popup],
})
export class App {
  constructor(injector: Injector, public popup: PopupService) {
    const PopupElement = createCustomElement(Popup, {injector});
    customElements.define('popup-element', PopupElement);
  }
}
